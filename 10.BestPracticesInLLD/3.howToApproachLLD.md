### How to approach a LLD questions

#### clarify requirements (3-5 mins)
- functional requirements
- non-functional requirements
- highlights edge cases and real world behaviour the system must handle.

#### Identify core entities (5-10mins)
- define all core domain entities relevant to system.
- Include attribute, responsibilities and relationship between entities
- Mention auxiliary/supporting entities if required

#### Visualize interaction flow (example bookMyShow-> search movie , lock, book a  movie) (5 min)
- describe who interacts with whom during interactions
- Use sequence flows , flowcharts to show
  - system interactions with user 

#### Define class structure and relationships (10-20 mins)
- Use OOP and SOLID principles
- Include
  - interfaces, abstract classes and concrete implementation
  - domain models, services gateways | adapters, DTOs(models)
- Apply design principles for scalability and extensibility
  - large coupling, high cohesion
  - abstraction for third part integerations
  - modularity for business logic
- prepare the design for production scale

#### Define core use cases and methods (10-20 mins)
- for every major features, define
  - method responsibilities
  - i/o models
  - collaborating classes
  - transformation flow
- Include use cases like
  - create, update, delete , fetch operations
  - real-time flows (seat locking, order status updates)

#### Apply design Patterns
- mention clearly which design pattern and why
- reinforce adherence to clean code practive

#### Handle edge cases
- discuss edge cases, failure scenarios and system limits
  - concurrency issues
  - stale state
  - partial failures
  - retry handling and idempotency
- discuss mitigation strategies like
  - locks, cache invalidation, compensation
  - consistency vs availability trade-off
